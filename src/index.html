前端组件库 Void-UI

  • 地址： https://github.com/HuijiFE/void-ui
  • 说明：基于 Vue 与 TypeScript 的前端 UI 组件库，服务于公司内部多个项目。
  • 职责：独立完成开发，与主设计师共同制定设计方案。
  • 技术：Vue, TypeScript

Void-UI，不同于主流面向后台管理系统的开源UI，是一个适合面向用户网站的前端 UI 库。根据集团的前端技术栈，选择了 Vue。同时我个人对于代码质量的高要求，使用了有 C# 基因 TypeScript 作为开发语言。伴随多个下游项目持续迭代，在业务中出现的许多可复用组件被抽离合并到该 UI 库中。Vue 的官方最佳实践是采用了自有的模板语法并不适合 TypeScript，为了更好地适配 TypeScript，改为使用 class component，吸收了 React 的 JSX 代替原本的模板语法。使用了当时较新的 CSS Flex 标准实现了布局组件，能够应对任意百分比的比例。 语义化的配色，颜色主题上有明暗两套配色，并能实时切换。封装了响应式媒体查询的 JS API。

Story：2017年7月我入职灰机维基，是创业公司，并于10月并入多牛传媒集团。 集团同时收购了多家游戏媒体网站，有重做多个网站的需求，我们团队需要承担接下来的多个旧网站改造和新网站任务。 在国内的前端社区中已经有了许多优秀的开源 UI，比如 Element-UI 和 Ant-Design，但在考察之后发现，这些项目更适合管理后台类的项目，并不适合面向用户的更有设计感的网站。 于是我们决定自己设计和开发一套通用 UI。 在前端领域，我虽然一个新手，前端核心的编程语言上也是边学习边开发。但 WPF 的开发经验给了我自信，即使团队里只有我一名前端，我有信心接下这个挑战。 在有限的时间中学习了大量前端必备的技能，打包工具 Webpack、测试工具 Jest，在团队的运维后端的指导下应用了 Jenkins 持续集成、 Docker 容器部署。 同时利用该项目培训了两位新人。

电玩巴士 游戏站

  • 地址： https://game.tgbus.com/
  • 说明：新项目，游戏资料库
  • 职责：独立完成前端开发，部分后端 API 开发，后期接手 UI 设计改版工作。
  • 技术：Vue, TypeScript, GraphQL, SSR, K8S, Void-UI

使用了前后端分离与服务端渲染技术，前端基于 Vue 和 TypeScript，API 使用了 GraphQL 技术，K8s 和 Jenkins 作为持续集成工具。 网站的功能是检索各大游戏平台的数万个游戏的各类信息，在计划中，是作为作为基础功能服务其他改造后的旧网站。 前端部分由我独立开发，是 Void-UI 的第一次落地实践。项目初期我承担了 MongoDB RESTful API 部分的开发。在不断迭代的过程中，面对大量的接口改动需求，前后端有大量重复性劳动和协同成本，我提出了 GraphQL，将 API 定义工作从后端转移到前端。后端同事采纳了我的建议改用了 GraphQL，并由后端使用 Go 语言 和 MySQL 进行后续的数据库开发。作为一个依赖搜索引擎的垂直网站，应对 SEO 优化的需求，我在中期完成 Node.js 服务端渲染的改造。同时全站 CDN 化，使用了部分 PWA 技术对缓存进行持续化，减少了网络流量的支出。

作为副产品，我封装了“GraphQL与TypeScript间类型”、“服务端渲染中间件”等多个基础复用模块，并用于后续的项目中。

RimTrans

  • 地址： https://github.com/RimWorld-zh/RimTrans
  • 说明：个人开源项目，开发于2016年。基于 .NET WPF 开发的桌面 GUI 应用，游戏 RimWorld 的翻译工具。目前正在使用 Electron 重构。
  • 技术：.NET WPF, Electron, TypeScript, Vue

因为业余爱好参与了游戏翻译，需要一个趁手的工具，于是在16年8月决定零基础自学编程。
初期使用 C++ 开发，于10月在 GitHub 上发布了第一个命令行版本。
在考虑到需要便于普通用户使用，有了 GUI 的需求，考察了 C++ 的 QT、C# 的 WinForm 和 WPF 后，选择了 C#/.NET/WPF 作为技术栈重新开发，开始了磕磕碰碰地一边学习 C#.NET 一边开发。在中文资料匮乏的情况下，参考了几个优秀的开源项目，遇到问题时在 StackOverflow 上查找英文资料。于17年1月发布了第一版带界面的版本。
继续深入学习 C# 和 WPF 并掌握了 Linq 和 MVVM 等一系列后，对项目进行重构，于3月发布了新版，拥有更美观的界面、友好的用户交互。

这个早期的个人项目相对于真正的商业项目来说，最多只是个玩具，但对于我个人来说，意义重大，让我真正进入了编程的世界，为日后的前端工作打下了坚实的基础。也在这过程中认识了灰机维基的创始人，并被直接聘用。
